# Product backlog

Tasks are grouped into product epics. Each task owns a `work/` scratchpad while
it is active and a `work/RESULTS.md` evidence summary before completion.

| Epic                                                                                   | Outcome                                                              | Status  |
| -------------------------------------------------------------------------------------- | -------------------------------------------------------------------- | ------- |
| [E00 — Issue triage and editing](epics/E00-issue-triage/README.md)                     | Users can fully create, find, triage, edit, and safely delete issues | done    |
| [E01 — Issue discussion](epics/E01-issue-discussion/README.md)                         | Teammates can discuss work directly on an issue                      | done    |
| [E02 — Powerful issue views](epics/E02-powerful-issue-views/README.md)                 | Users can build, save, and share trustworthy issue views             | pending |
| [E03 — Multiple workspaces](epics/E03-multiple-workspaces/README.md)                   | Users can connect and switch workspaces without mixing data          | pending |
| [E04 — Bulk issue triage](epics/E04-bulk-issue-triage/README.md)                       | Users can process many issues safely and efficiently                 | pending |
| [E05 — Cycles and sprint planning](epics/E05-cycles-and-sprint-planning/README.md)     | Teams can plan and manage time-boxed work                            | pending |
| [E06 — Project planning and roadmap](epics/E06-project-planning-and-roadmap/README.md) | Users can manage projects and understand delivery plans              | pending |
