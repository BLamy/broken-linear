# E02-T03 — Saved and shareable views

Status: `pending`

## User outcome

Users can return to useful issue views and share the exact same view with a
teammate.

## Acceptance criteria

- Filter, sort, and density state round-trips through the URL.
- Browser back, forward, refresh, and direct links restore the same view.
- Users can save, name, rename, apply, and delete personal views.
- Shared views explain unavailable criteria rather than silently changing them.
- Each workspace restores its own last-used and saved views.

Create `work/` when implementation starts and record evidence in
`work/RESULTS.md`.
