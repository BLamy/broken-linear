# E02-T02 — Multi-select filters and chips

Status: `done`

## User outcome

Users can combine several filter values and always see why an issue list looks
the way it does.

## Acceptance criteria

- Status, priority, assignee, project, team, and label support multiple values.
- Values within a category use inclusive matching; categories compose together.
- Active criteria appear as keyboard-operable removable chips.
- Clear-one and clear-all update the result count immediately.
- Opening and editing filters does not unexpectedly close the workflow.

Create `work/` when implementation starts and record evidence in
`work/RESULTS.md`.
