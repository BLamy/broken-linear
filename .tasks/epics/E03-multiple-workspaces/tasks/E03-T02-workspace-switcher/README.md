# E03-T02 — Workspace switcher

Status: `pending`

## User outcome

Users can move between workspaces quickly and always know which workspace is
active.

## Acceptance criteria

- The sidebar header opens a keyboard-accessible workspace switcher.
- Every row shows workspace name, icon or initials, and active state.
- Switching updates the URL and restores that workspace's last visited view.
- Loading states never show one workspace's data under another workspace's name.
- Switching failure preserves the current workspace and offers retry.

Create `work/` when implementation starts and record evidence in
`work/RESULTS.md`.
