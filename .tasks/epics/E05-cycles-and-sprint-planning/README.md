## Release criteria

- Current, upcoming, and recent cycles are navigable per team.
- Cycle pages distinguish active, completed, upcoming, and no-cycle states.
- Progress and issue counts reflect persisted issue state.
- Issue creation, details, and cycle views support valid assignment changes.
- Replay QA proves the full planning lifecycle across reloads.
