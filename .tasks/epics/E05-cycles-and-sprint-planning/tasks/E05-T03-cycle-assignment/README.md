# E05-T03 — Cycle assignment

Status: `pending`

## User outcome

Users can plan issues into cycles and adjust the plan as work changes.

## Acceptance criteria

- Issue creation and details support cycle assignment and removal.
- Cycle views can move issues between current and upcoming cycles.
- Only cycles valid for the issue's team are offered.
- Invalid cross-team assignments are blocked with a clear explanation.
- Changes update issue and cycle progress without a hard reload.

Create `work/` when implementation starts and record evidence in
`work/RESULTS.md`.
