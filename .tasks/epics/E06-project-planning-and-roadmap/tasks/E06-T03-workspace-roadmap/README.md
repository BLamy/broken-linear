# E06-T03 — Workspace roadmap

Status: `pending`

## User outcome

Users can scan delivery plans across teams and open the project that needs
attention.

## Acceptance criteria

- The roadmap groups projects by status and target window.
- Users can filter by team, lead, status, and date range.
- Project cards expose progress, health, owner, and target timing.
- Every card links to its project overview.
- Progress and target-date changes update without a hard refresh.

Create `work/` when implementation starts and record evidence in
`work/RESULTS.md`.
