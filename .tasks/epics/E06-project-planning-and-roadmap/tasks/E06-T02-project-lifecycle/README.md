# E06-T02 — Project lifecycle

Status: `pending`

## User outcome

Authorized users can create and maintain projects without leaving the app.

## Acceptance criteria

- Users can create and edit project name, description, dates, teams, lead, and status.
- Validation and mutation failures preserve entered values.
- Archive requires confirmation and removes the project from default navigation.
- Archiving does not orphan or delete project issues.
- Archived projects can be found and restored.

Create `work/` when implementation starts and record evidence in
`work/RESULTS.md`.
