# E06-T01 — Project overview

Status: `pending`

## User outcome

Users can understand a project's ownership, timing, health, scope, and progress
without reconstructing it from individual issues.

## Acceptance criteria

- Project pages show description, status, lead, target date, and teams.
- Issue totals and completion progress reflect current project issues.
- Overdue and at-risk states do not rely on color alone.
- Missing optional metadata has deliberate empty states and edit affordances.
- Project issue filters and links preserve project context.

Create `work/` when implementation starts and record evidence in
`work/RESULTS.md`.
